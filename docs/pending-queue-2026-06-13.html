9b — StoreProduct + Product + ProductImage (catalog/inventory cluster) — investigation DONE 2026-06-13

The most intricate part of the system — core daily merchant workflows (add/edit product, inventory). Each save is a multi-table transaction. Build it in a dedicated session; do NOT rush (breaking these = merchants can't manage their catalog/stock).
Write flows (verified by reading each):
  • AddCustomProductModal (grocery): Product insert/update → StoreProduct insert/update (price,stock) → ProductImage delete+insert. Storage uploads before.
  • AddMenuProductModal (dining): Product upsert → ProductImage delete+insert → StoreProduct delete(if edit)+upsert with VARIANTS (onConflict branch_id,productId,variant).
  • useInventory: updateItem (StoreProduct update price/stock/active) + deleteItem (StoreProduct soft-delete is_deleted=true). Also inserts notifications (separate table — leave).
  • ConfigureProductsModal: bulk StoreProduct upsert (onConflict branch_id,productId,variant) — configure existing catalog items for a branch.
  • admin StoreProductTable.tsx:73: StoreProduct update.
Proposed endpoints (writes via supabaseAdmin/Prisma; auth: userCanManageBranchFull on the StoreProduct branch_id, or admin). All additive — safe to build+deploy before refactoring:
