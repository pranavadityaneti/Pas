E · Phase 7G residuals (audit mediums/lows) DEFERRED

  1. Negative-net cycle UI semantics — clawback-only weeks. Founder decision needed: carry debt forward, invoice the merchant, or block payment until offset.
  2. Refund rupee-rounding vs clawback paise — refund path rounds whole rupees before paise; recorded clawback can differ < ₹1 from moved money. Align rounding.
  3. Close scalability — single IN-list scan from epoch; needs paging before thousands of orders/week.
  4. Missed-Monday catch-up — if process down at 02:00 IST, close skipped. Add startup catch-up check.
  5. detectClawback DB uniqueness — currently find-then-create; add a defensive partial unique index.
  6. Order-create price validation — items still client-priced. Coherence guard catches it at close-time (HELD + Sentry); root fix is validating against StoreProduct at create. Hot path — schedule deliberately.
  7. Payment-verification override UI — endpoint exists (POST /admin/orders/:id/payment-verification, audit-logged); admin worklist UI not built. Until then: curl with admin JWT.
  8. Residual micro-race — refund whose tx commits between in-tx revalidation and close commit can settle full with no clawback (sub-second window). Sentry mid-close drop messages + weekly reconciliation cover it; full fix needs SELECT FOR UPDATE on candidates.
  9. Negative-net + mark-paid semantics — flow currently treats it like a payout; needs UI guard + founder decision (#1 above).
