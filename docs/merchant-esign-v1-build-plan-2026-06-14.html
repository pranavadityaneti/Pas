Architecture — single source of truth

  • Agreement content = structured data (sections/clauses per vertical), rendered two ways: React Native components for the in-app read view, and an HTML string for the expo-print PDF. This avoids needing a WebView in the app.
  • Signature = SVG path (gesture-handler captures strokes → react-native-svg renders). Embedded inline in the PDF HTML and stored as vector — no rasterization, so no extra native dependency.
  • Legal record → new merchant_consents table; signed PDF stored in a private merchant-agreements bucket.
  • Attribution = Merchant Details header (top) + Execution block (end), both merge-filled from data already captured at signup.
