4. OTA discipline going forward

  1. Right now the OTA queue is empty — the submitted builds carry all of the above. Don't push an OTA until 1.1.3 / 1.2.6 are live, or you'll be updating a build no one has yet.
  2. Pre-OTA .env protocol: eas update reads the local .env (builds use the eas.json profile env). Before any OTA: confirm EXPO_PUBLIC_API_URL=https://api.pickatstore.io, push with --clear-cache, grep the bundle, then revert.
  3. runtimeVersion = appVersion: an OTA only reaches the matching version. After these go live, OTAs target 1.1.3 / 1.2.6; older installs must update via the store.
  4. One app at a time, verify each (your stated protocol): eas update --branch production --message "<what changed>", confirm on a device, then the next.
  5. Anything touching a native module / app.json / a new dependency = NOT an OTA → it needs the next build.
