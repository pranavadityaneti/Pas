Theme 1 — Order / payment state machine (P0 cluster)

Three shared root causes drive six bugs:

  • SRC-A "two clocks": the customer's countdown uses the server expires_at, but the actual auto-expire is a client setTimeout(120s) started later (after the network round-trip). They never reconcile → timer drift + premature "timed out". (useOrderRequests.ts:50,183-188)
  • SRC-B coupled multi-store gate: allResolved = requests.every(status !== 'PENDING') — the screen won't advance until all stores answer, so an accepted store waits on a slow sibling. (useOrderRequests.ts:61)
  • Load-bearing defect: the server cron expireStaleOrderRequests expires status IN ('PENDING','ACCEPTED') past expires_at — so an accepted-but-unpaid order is killed at 2 min and shows "timed out". (scheduled-jobs.ts:571-585)
