How the admin dashboard ships (important): unlike the mobile apps, admin-web has no build/OTA cycle — it deploys to Vercel from git. A push to the feature branch = a Vercel preview only; production updates on a merge to main. So the items below are built + committed; each is live in production once its commit is on main. 1. Login & Auth

  1. NEW WhatsApp OTP login as the primary sign-in, with email/password as a fallback (Login.tsx, AuthContext.tsx).
  2. IMPROVED Login redesign — frosted photo backdrop, phone-OTP-first flow.
  3. NEW Role assigned on OTP login — the admin's role resolves at sign-in and drives the whole UI.

2. RBAC, Roles & Admin Invites

  1. NEW RBAC v1 — role model + role-aware sidebar (each role sees only its modules) (lib/rbac.ts, Sidebar.tsx).
  2. NEW Roles & Permissions module (RolesPermissions.tsx).
  3. NEW Invite Admin flow — phone + email invites (AddAdminDialog.tsx).
  4. NEW Inline role/status edit for existing admins.

3. Role-aware Home Dashboards

  1. NEW Per-role home pages — Super-Admin, Finance, Operations, Support each get a tailored dashboard (home/SuperAdminHome.tsx, FinanceHome.tsx, OperationsHome.tsx, SupportHome.tsx, _shared.tsx).
  2. NEW Home KPI stat endpoints wired to real data.
  3. IMPROVED Quick Actions moved to the header (beside the date chips) instead of the page bottom.

4. Analytics / Reports

  1. NEW Real analytics charts (replaced demo data) (analytics/AnalyticsDashboard.tsx, AnalyticsHub.tsx).
  2. NEW Analytics v1 — date filters, KPI strip, CSV export, sortable stores.
  3. IMPROVED Analytics depth — compare-vs-previous-period, top products, by-category, by-city breakdowns.
  4. NEW Audit log view + helper (admin action history) (analytics/AuditLog.tsx).
  5. FIX Renamed /analytics → /reports; removed the cyan→blue gradient for design consistency.

5. Customers

  1. NEW Customers database on real data with Block / Unblock (customers/CustomerDatabase.tsx, useCustomers.ts).
  2. NEW Customer detail drawer — real addresses (drawer + hover), order history (CustomerDetailsSheet.tsx).
  3. NEW Edit customer name, incomplete-signup toggle, duplicate-account hint.
  4. NEW Deep link /customers?id=<userId> opens the detail drawer directly.
  5. FIX Expanded filters; clean name/email display; full_name read-fallback + name backfill.
  6. FIX Reads routed through the admin API (Prisma) instead of direct Supabase (fixed PostgREST schema-cache misses) + pagination.

6. Orders

  1. NEW Order Manager on the real Order table with the real status enum (dropped demo dressing) (orders/OrderManager.tsx, useOrders.ts).
  2. FIX Covers all 10 OrderStatus values + restored realtime subscription (orders update live).
  3. FIX Order reads routed through the admin API (Prisma).

7. Merchants & KYC

  1. NEW KYC review queue — shows the real vertical name + full signup details; "Needs info" status to bounce incomplete applications back (merchants/KYCQueue.tsx).
  2. NEW Merchant directory + detail sheet (MerchantDirectory.tsx, MerchantDetailsSheet.tsx).
  3. NEW Add / Edit merchant, Export merchants, Merchant report dialogs (AddMerchantSheet.tsx, EditMerchantDialog.tsx, ExportMerchantsDialog.tsx, MerchantReportDialog.tsx).
  4. NEW Store product table — view a merchant's listed inventory (StoreProductTable.tsx).
  5. FIX Phase 2 FINAL B7 — store inventory read via branch_id (the corrected key).
  6. FIX All merchant_branches writes routed through the API (Phase 8) (useMerchants.ts).

8. Master Catalog

  1. NEW Master Catalog module — Global catalog, Merchant Requests, Sync Queue tabs (catalog/MasterCatalog.tsx).
  2. FIX Taxonomy fix (the blank-columns bug you spotted) — the Category/Vertical dropdowns now load live data (15 Verticals / 136 Tier2Categories) instead of stale hardcoded lists, so correctly-filed products no longer render blank. Columns relabelled Category (= Vertical) and Subcategory (= Tier2Category) to match the customer app; subcategory cascades by category.

9. Marketing & Coupons

  1. NEW Coupon engine UI — full Coupon Builder with live preview, brand theming, preset theme picker (marketing/CouponBuilder.tsx, CouponCard.tsx).
  2. NEW Coupon detail page + Coupon analytics hub (redemptions, performance) (CouponDetailPage.tsx, CouponAnalyticsHub.tsx, MarketingHub.tsx).
  3. NEW Merchant-signup coupons CRUD — manage partner coupons (MerchantSignupCoupons.tsx, lib/merchantSignupCouponService.ts).
  4. IMPROVED Coupon foolproof Phases 1–3 — schema + server hardening + admin analytics/audit wiring (lib/couponService.ts).

10. Finance / Settlements

  1. NEW Settlements Manager — merchant payout/settlement view (finance/SettlementsManager.tsx, lib/settlementService.ts).

11. Operations

  1. NEW Two-way Wati support inbox — threaded WhatsApp conversations with customers + reply from the dashboard (operations/CustomerSupportInbox.tsx).
  2. NEW Refunds & Disputes queue — surfaces stranded/pending payments + guarded (controlled) refunds (operations/RefundsDisputes.tsx).

12. Global Config

  1. NEW Global Config panel — edit platform settings (service radius, minimum-order value) that the customer app reads live (analytics/GlobalConfig.tsx → platform_settings).

13. Design system, Monitoring & Architecture

  1. IMPROVED Admin design system — cross-module UI alignment to a shared design language (docs/admin-design-system.md); refreshed logos/login background.
  2. NEW Sentry monitoring wired into admin-web (error tracking).
  3. FIX Architecture shift — Customers/Orders/Merchants reads + all branch/Store/catalog/inventory writes routed through the admin API (Prisma) instead of direct Supabase (Phases 8/9), fixing schema-cache misses + centralising auth.
  4. FIX Secured API auth interceptor; Vercel production proxy config; .map crash patch.
